Access example datasets, specify global settings, import raw SIMS data, save intermediate results, or open previously analysed datasets.
Examples:
A pull-down menu with five pre-processed example datasets, including four Cameca 1280HR datasets from the Institute of Geology and Geophysics (IGG) at the Chinese Academy of Sciences in Beijing, and one SHRIMP-II dataset from Geoscience Australia (GA). Two of the datasets contain stable isotope data (for O and S) and the remaining three contain U-Pb and U-Th-Pb data. Choose the most appropriate dataset before uploading your own data.
Description:
A one sentence summary of the origin and purpose of the data.
Instrument:
Either 'Cameca' or 'SHRIMP'.
Ions:
A comma-separated list of names for the different mass channels analysed. Atomic ion channels must be named as a chemical symbol followed by an isotope number (e.g., 'O16', 'U238'). If the column headers in your data file do not follow this format, then you can rename them here.
Numerators:
A comma-separated list of names for the numerators of the isotopic ratios that are to be processed. The names must occur in the list of Ions shown above.
Denominators:
A comma-separated list of names for the denominators of the isotopic ratios that are to be processed. The list must contain the same number of items as the Numerators field, and may only contain names that occur in the list of Ions shown above.
Blank:
Either 'nominal' (for Cameca data) or, for single-collector datasets, the name of a channel in the Ions field, or a numerical value with the background signal in counts per second.
Load raw SIMS data:
Here you can upload your own raw SIMS data. For Cameca data, select multiple .asc files. For SHRIMP data, select a single .op or .pd file. Support for .xml files will be added later.
Save:
Save the data, settings and processing steps in a .json file for future reprocessing. This function can be used to store your own simplex 'template'.
Open:
Open a .json file with previously analysed data and settings. This function can be used to open your own simplex 'template'.
